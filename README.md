# ai-native-learning (AI Native Learning)

[中文](README.zh-CN.md)

`ai-native-learning` is a video-first learning system: load a video URL or local media file, play it in the browser, and get streaming bilingual captions generated by the backend pipeline (**download/transcode -> transcribe -> translate**).

## Core Reuse Strategy

This project intentionally reuses mature modules from the monorepo:

| Capability | Reused module |
|---|---|
| Video download + playable mp4 conversion | `scripts/lib/video_download.py::download_and_transcode` |
| VAD segmentation + cloud STT streaming | `scripts/lib/transcript_common.py` + `scripts/audio_to_transcript.py` |
| Batch LLM translation (SSE + cache) | `subtitle_player/backend/translator.py` |
| Unified model config (settings panel JSON) | `ai_native_learning/backend/model_config.py` |
| Subtitle models + source language detection | `subtitle_player/backend/models.py`, `subtitle_player/backend/parser.py` |

## How It Works

```text
URL -> yt-dlp download + ffmpeg transcode -> local playable.mp4 -> immediate <video> playback
                         |
                         -> ffmpeg VAD chunks -> cloud STT (streaming) -> source captions --+
                                                                                              +-> SSE to frontend
                                      LLM batch translation (streaming) -> target captions ---+
Frontend: <video>.currentTime drives active segment matching and subtitle rendering.
```

- Video is playable as soon as media preparation finishes.
- Transcription and translation continue in parallel, and subtitle segments stream incrementally.
- Re-submitting the same source reuses cached artifacts (`<id>.vtjob.json`, `<video>.bilingual.json`).

## Quick Start

```powershell
# Run from repository root (dependencies are in root pyproject.toml)
uv run python ai_native_learning/run.py
```

Open `http://127.0.0.1:8810`.

Input options:
- Online video URL (YouTube/Bilibili/etc.)
- Local media file path
- Local file upload via the UI

Supported local formats:
- Video: `mp4,mkv,mov,webm,avi,flv,m4v,ts,wmv`
- Audio: `mp3,m4a,wav,flac,ogg,aac,opus`

CLI flags:

| Flag | Default | Description |
|---|---|---|
| `--host` | `127.0.0.1` | Bind address |
| `--port` | `8810` | HTTP port |
| `--no-open` | off | Do not auto-open browser |

## Configuration

**Configure only via the settings panel (⚙)** — no `.env` fallback; missing config causes the related feature to fail with an error.

Storage: `%APPDATA%/AiNativeLearning/models_config.json` (empty `{}` on fresh install).

### Text models (LLM)

- Global: `base_url`, `api_key`, `temperature`, `max_tokens`
- Per-task models: `translate`, `outline`, `highlight`, `quiz`, `grade`, `feynman`, `report`

### Transcription (STT)

Three backends can be saved; only the **active** one is used at runtime (no cross-backend fallback):

| Backend | Description |
|---|---|
| `whisper` | Local Whisper service URL |
| `cloud` | Cloud STT (OpenAI-compatible) |
| `funasr` | Fun-ASR + OSS credentials |

Media and subtitle cache: `%APPDATA%/AiNativeLearning/media/`.

### Fallback download proxy

In settings, **Download proxy** tab: when enabled, yt-dlp retries via the configured proxy (e.g. `socks5h://127.0.0.1:12080`) after a direct failure/timeout for both **video download** and **metadata fetch** (title/id). When disabled, only direct download is used (no hardcoded proxy).

Required tools in PATH: `yt-dlp`, `ffmpeg`, `ffprobe`.

## API Overview

| Endpoint | Description |
|---|---|
| `POST /api/job` | Submit source URL or local path (`{source}`) |
| `POST /api/job/upload` | Upload local media file |
| `GET /api/job/{id}` | Get current job snapshot |
| `GET /api/job/{id}/events` | SSE events (status/progress/segments/translations/done) |
| `GET /api/video/{id}` | Range-enabled local mp4 serving |
| `GET/PUT /api/models-config` | Read/write model configuration |
| `POST /api/models-config/reset` | Clear configuration |
| `POST /api/models-config/test/text` | Test text model (`{ task }`) |
| `POST /api/models-config/test/stt` | Test transcription backend (optional `{ backend }`) |

## Known Limitations

- First run for a new video may take time (download + transcode + initial STT).
- Timeline alignment is segment-level (VAD based), not word-level forced alignment.
- Requires network access to the source video platform (with optional SOCKS5 fallback in `yt-dlp`).
